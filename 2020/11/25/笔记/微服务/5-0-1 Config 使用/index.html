在Github/码云 上 新建一个项目

clone 到本地 然后在该项目新建几个配置文件并提交（文件格式必须是utf-8格式的）. 根据需要建。

config-dev.yml
config-prod.yml
config-test.yml

项目中使用

配置服务端

pom文件

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-config-server</artifactId>
 </dependency>

application.yml

spring:
  application:
    name: cloud-config-center
  cloud:
    config:
      server:
        git:
          uri:  https://github.com/weimao520/SpringCloudConfig.git  #配置中心地址
		  search-paths:  # 搜索路劲
            - spring-cloud-config
      label: master # 分支

启动类

@SpringBootApplication
public class ConfigCenterMain{

    public static void main(String[] args) {
        SpringApplication.run(ConfigCenterMain.class, args);

    }
}

// 然后启动项目 master 分支 / config-dev.yml 文件名

访问 localhost:8081/master/config-dev.yml 就可以访问其GitHub配置信息

http://localhost:8081/config/dev/master 这个也行，具体的请看官网

编写客户端

pom

相比较服务端 结尾少了 server

spring-cloud-starter-config-server


<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-config</artifactId>
</dependency>

bootstrap.yml

ps：application.yml 是用户级资源配置项 bootstrap.yml 是系统级优先级更高

spring:
  application:
    name: config-clent
  cloud:
    config:
      label: master
      name: config  # 配置文件名称
      profile: dev # 读取后缀名称  三个综合(label,name.profile)  在master分支上config-dev.yml的配置文件 地址是http://config-8081.com:3344/master/config
      uri: http://localhost:8081 # 配置中心地址


## 要想实现动态配置 配置如下（就是更新GitHub上的配置文件，如果不配置的话需要重新启动该服务）
management:
  endpoints:
    web:
      exposure:
        include: "*"

启动类

@SpringBootApplication
public class ConfigClientMain80 {

    public static void main(String[] args) {
        SpringApplication.run(ConfigClientMain80.class, args);
    }
}

测试访问 编写Controller

//RefreshScope  动态刷新配置
@RestController
@RefreshScope
public class ConfigClientController {

    @Value("${config.info}")
    private String configInfo;

    @GetMapping("/configInfo")
    public String getConfigInfo() {
        return this.configInfo;
    }
}

这样的话 还是不行 ；需要 访问 http://localhost/actuator/refresh 发送post 请求服务端的配置

我是使用curl

curl -X POST "http://localhost/actuator/refresh"

存在 问题

如果配置多个微服务，每个服务都要执行一次post请求。
