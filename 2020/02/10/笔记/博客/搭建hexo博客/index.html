说明

​ 使用hexo搭建个人博客是一个朋友推荐的；因为可以说是完全免费，虽然我自己也有服务器，但是还是使用了这个，要问我为什么，就是搭建简单。ps : 程序员最缺的就是时间不是么 :smiley:

环境配置

  • ​ 下载 node js
  • ​ 下载 git Windows系统的 可以下载淘宝镜像比较快

安装hexo

npm install -g hexo-cli
hexo -v  //  检验是否安装成功

配置博客

//  创建myblog
mkdir myblog
//  进入 博客目录
cd myblog 
// 初始化
hexo init

  • node_modules: 依赖包
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章 一般自己写的笔记 丢到 source/_posts/ 下就好了
  • themes：主题 ## 这个自己找（看看自己对页面功能的需求），用的最多的是 next 我用的是 fluid
  • ** _config.yml: 博客的配置文件**
hexo g  //  编译
hexo s  //  本地启动 默认4000 端口

把博客配置到GitHub ，这样所有人都能访问的到

前提：个人必须要有一个GitHub的账号

  1. 去 GitHub 新建一个工程 （要求： 格式必须是 用户名.github.io 比如 name.github.io）

工程名必须 跟自己名称一致 加上 .Github.io  **我这里报错 的是因为之前创建了**

![image-20200726154608047](D:\笔记截图\博客\image-20200726154608047.png)

  1. 然后去我们创建的博客根目录下 修改 _config.yml问价 找到 deploy 这个部分, 修改成

    deploy:
    type: git
    repo: git@github.com:xxx/xxx.github.io.git  ###  你在GitHub创建完工程 生成的地址
    branch: master

  2. 部署

    hexo  cl  ##清理

hexo  g  ## 编译   ##  每次操作完最好 执行一次，不然会发现你本地运行好好的，远程的内容跟你本地不一致

npm install hexo-deployer-git --save   ## 安装git环境  

hexo d  ##部署到GitHub  

 ##  他会让你输入GitHub的账号密码 输入一下就好了

这一步就完成了 然后试试访问一下部署的地址就好了 如：xxx.github.io 我的就是：weimao520.github.io 如果有点闲钱的话 最好自己买一个域名 配置下:smiley:
